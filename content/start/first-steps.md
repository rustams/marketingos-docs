# Как начать работу после установки

После установки MarketingOS начните с короткого сценария: создайте первую кампанию, добавьте к ней несколько карточек контента и проверьте работу статусов.

Так вы сможете понять логику рабочего места и подготовить его к работе команды.

> Место для скриншота: первый экран после установки или кнопка перехода в рабочее место MarketingOS.

## Перед началом

Перед созданием первой кампании определите:

- какую маркетинговую активность нужно вести;
- кто будет ответственным за кампанию;
- какие материалы нужно подготовить;
- какие сроки важно учитывать;
- какие результаты нужно зафиксировать после публикации.

Для первого запуска лучше выбрать одну понятную кампанию. Не переносите весь маркетинговый процесс компании сразу.

## Шаг 1. Откройте маркетинговое рабочее место

Перейдите в рабочее место MarketingOS на портале Bitrix24.

Проверьте, что доступны основные разделы:

- Кампании;
- Контент;
- Медиафайлы.

Если разделы не отображаются, проверьте, завершилась ли установка приложения и есть ли у пользователя права на просмотр рабочего места.

> Место для скриншота: навигация по рабочему месту с тремя основными разделами.

## Шаг 2. Создайте первую кампанию

Откройте раздел «Кампании» и создайте новую кампанию.

Заполните основные данные:

- название;
- цель;
- сроки;
- ответственный;
- описание.

Название должно быть понятным для команды. Например: «Запуск весенней рассылки» или «Контент-план на июнь».

> Место для скриншота: создание или карточка кампании.

## Шаг 3. Добавьте карточки контента

После создания кампании добавьте материалы, которые нужно подготовить.

Каждая карточка контента — это отдельный материал или единица работы.

Примеры карточек:

- пост для Telegram;
- статья для сайта;
- письмо для рассылки;
- рекламный баннер;
- презентация;
- видео;
- вебинар.

Для каждой карточки укажите:

- название материала;
- формат;
- ответственного;
- срок;
- связь с кампанией;
- краткое описание задачи.

> Место для скриншота: список карточек контента внутри кампании или отдельная карточка контента.

## Шаг 4. Добавьте медиафайлы, если они нужны

Если для материала используются изображения, видео, документы или другие файлы, создайте карточки медиафайлов.

Это поможет сохранить материалы в общей структуре и зафиксировать условия их использования.

В карточке медиафайла можно указать:

- название;
- тип материала;
- источник;
- условия использования;
- связь с карточкой контента или кампанией.

Если на первом запуске медиафайлы не нужны, этот шаг можно пропустить.

## Шаг 5. Назначьте ответственных и сроки

Проверьте, что у кампании и карточек контента указаны ответственные.

Это нужно, чтобы каждый участник понимал:

- за что он отвечает;
- к какому сроку нужно выполнить работу;
- где смотреть статус задачи;
- куда вносить результат.

Если ответственный не указан, карточку сложнее контролировать.

## Шаг 6. Используйте статусы

Статусы помогают отслеживать движение работы.

Обновляйте статус карточки, когда работа переходит на следующий этап. Так руководитель и команда будут видеть текущее состояние процесса.

Обычно стоит проверять:

- что запланировано;
- что уже в работе;
- что готово к проверке;
- что опубликовано;
- что завершено.

Точная логика статусов зависит от установленной версии конфигурации.

> Место для скриншота: карточка со статусом или список карточек по статусам.

## Шаг 7. Зафиксируйте публикацию

Когда материал опубликован, отметьте это в карточке контента.

Если доступны показатели, внесите их вручную.

Например:

- дата публикации;
- просмотры;
- переходы;
- заявки;
- другие показатели, которые важны для вашей команды.

В первой версии MarketingOS не получает эти данные автоматически из рекламных кабинетов или внешних площадок.

## Шаг 8. Завершите кампанию

Когда основные материалы опубликованы и результаты зафиксированы, завершите кампанию.

Перед завершением проверьте:

- все ли ключевые карточки закрыты;
- есть ли просроченные материалы;
- внесены ли доступные показатели;
- зафиксированы ли выводы по кампании.

В итогах кампании стоит указать, что получилось хорошо, где возникли задержки и что нужно учесть в следующей кампании.

## Минимальный сценарий для первого запуска

Для первого знакомства с MarketingOS достаточно пройти короткий маршрут:

1. Создать одну кампанию.
2. Добавить 2–3 карточки контента.
3. Назначить ответственных и сроки.
4. Перевести одну карточку по статусам.
5. Отметить публикацию вручную.
6. Внести один базовый показатель.
7. Завершить кампанию.

Такой сценарий позволяет проверить основную логику без переноса всей маркетинговой работы в систему сразу.

## На что обратить внимание на старте

### Размер первой кампании

Для первого запуска лучше выбрать небольшой сценарий. Так будет проще проверить процесс и понять логику рабочего места.

### Ответственные

У кампании и карточек контента должны быть ответственные. Это помогает команде понимать, кто двигает работу дальше.

### Статусы

Статусы нужно обновлять по ходу работы. Если их не использовать, руководителю будет сложнее контролировать текущее состояние кампании.

### Результаты

После публикации стоит фиксировать хотя бы базовые показатели. Даже простые ручные данные помогают оценить итог кампании.
